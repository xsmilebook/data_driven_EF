## APP导出问题描述
对于APP数据的导出，存在以下问题：
1. 被试进行任务时都使用设备本地存储的正式阶段刺激图片/Item名与正式阶段正确答案。但导出时正式阶段正确答案，正式阶段被试按键，相对时间(秒)等信息都是由执行任务所在设备保存的txt文件导出，但是正式阶段刺激图片/Item名信息由网站后台导出；
2. 在数据采集的过程中，更换了几次正式阶段刺激图片/Item名和正式阶段正确答案序列，更换序列的时间列在下方。在更换序列的过程中，可能会出现正式阶段正确答案设置错误的情况，所以正确答案不一定可靠（但大部分应该是正确的，少部分试次出错）。此外，由于采集数据时使用了多台设备，有可能某一台被试的序列与同一时段其他被试的序列不一样（可能表现在因为下载时间一样所以刺激序列一样，但正确答案序列不一样）；
3. 由于网站后台的问题，下载的数据都会使用网站当前存储序列的正式阶段刺激图片/Item名，也就是说如果2025/11/29被试的数据（本地测试时使用visit2序列）在2025/12/20下载（网站当时使用了visit1序列），那么该被试的正式阶段刺激图片/Item名是visit1序列（网站当时存储），正式阶段正确答案是visit2序列（设备本地存储txt）；
4. 在项目进行过程中，网站曾出现崩溃。在这段时间中（2024/12至2025/02，可能包括部分2025/06以及2025/07，和相当一部分2024的数据，一小部分2023年的数据）的被试数据（正式阶段正确答案，正式阶段被试按键，相对时间(秒)）由本地的txt文件导出，但正式阶段刺激图片/Item名选择了一个参照的被试（如果该参照被试是2025/07/08采集的，那么会导致本地的txt文件导出的被试序列变成visit3）。
